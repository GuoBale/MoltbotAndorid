# 常用 App 官方 API 调研

本文档汇总**微信、QQ、支付宝、淘宝、京东、美团、抖音、小红书、B站、百度地图、高德地图**等常用应用的**官方开放平台与 API**，便于判断哪些能力可通过官方接口实现、哪些只能通过深链接「打开到某页」。

---

## 一、结论速览

| 应用 | 是否有官方开放平台 | 官方 API 主要用途 | 能否「控制用户手机上的 App」 |
|------|---------------------|-------------------|------------------------------|
| 微信 | ✅ 有 | 登录、支付、公众号/小程序 | ❌ 不能 |
| QQ | ✅ 有 | 登录、获取用户信息、分享 | ❌ 不能 |
| 支付宝 | ✅ 有 | 支付、会员、生活号、小程序 | ❌ 不能 |
| 淘宝 | ✅ 有 | 淘宝客、千牛、商品/订单（卖家/ISV） | ❌ 不能 |
| 京东 | ✅ 有 | 京东联盟转链、商品、订单（推广者） | ❌ 不能 |
| 美团 | ✅ 有 | 外卖、企业用餐、配送（商家/ISV） | ❌ 不能 |
| 抖音 | ✅ 有 | 登录、视频发布、用户/粉丝数据、分享 | ❌ 不能 |
| 小红书 | ✅ 有 | 小程序、商家/服务商、商品与订单 | ❌ 不能 |
| B站 | ✅ 有 | 账号、视频稿件、直播（创作者/应用） | ❌ 不能 |
| 百度地图 | ✅ 有 | 地理编码、路线规划、POI（HTTP API） | ❌ 不能，但可替代部分「打开地图」场景 |
| 高德地图 | ✅ 有 | 地理编码、路线规划、POI（HTTP API） | ❌ 不能，同上 |

**共同点：**  
- 官方 API 都是给**开发者/商户/创作者**在自己的应用、网页或后台系统里用的。  
- **没有**「在用户手机上远程控制微信/支付宝/淘宝等 App、读聊天/订单/支付结果」的官方 API。

## 二、各平台官方入口与能力概览

### 1. 微信 (WeChat)

- **开放平台**  
  - 官网：<https://open.weixin.qq.com/>  
  - 文档：<https://developers.weixin.qq.com/doc/>
- **主要 API / 能力**  
  - 移动应用：微信登录、微信支付、分享到微信/朋友圈。  
  - 公众号：消息与事件、菜单、用户管理、素材等。  
  - 小程序：后端接口、云开发、硬件等。  
  - 第三方平台：代公众号/小程序管理。
- **面向谁**  
  网站/移动应用开发者、商户、服务商。  
- **与「控制用户手机微信」的关系**  
  仅能做「在你的 App 里调起微信登录/支付/分享」，不能读用户聊天记录或控制微信界面。

---

### 2. QQ / 腾讯开放平台

- **开放平台**  
  - 官网：<https://open.qq.com/>  
  - QQ 互联：<https://connect.qq.com/>  
  - API 列表：<https://wiki.connect.qq.com/api%E5%88%97%E8%A1%A8>
- **主要 API / 能力**  
  - QQ 登录（OAuth2.0）。  
  - 获取用户昵称、头像、性别等。  
  - 分享到 QQ、QQ 空间。
- **面向谁**  
  网站、移动应用开发者。  
- **与「控制用户手机 QQ」的关系**  
  仅能做「在你的 App 里用 QQ 登录、分享」，不能读 QQ 聊天或控制 QQ 客户端。

---

### 3. 支付宝 (Alipay)

- **开放平台**  
  - 官网：<https://open.alipay.com/>  
  - 文档：<https://opendocs.alipay.com/>
- **主要 API / 能力**  
  - 支付：APP 支付、手机网站支付、当面付、转账等。  
  - 会员：用户授权、用户信息。  
  - 生活号、小程序、第三方应用（ISV）。  
  - IoT、小游戏等。
- **面向谁**  
  商户、开发者、服务商。  
- **与「控制用户手机支付宝」的关系**  
  仅能做「在你的应用里调起支付宝付款、获取授权信息」，不能读用户账单或控制支付宝 App 界面。

---

### 4. 淘宝 (Taobao)

- **开放平台**  
  - 官网：<https://open.taobao.com/>  
  - 文档：<https://developer.alibaba.com/docs/api_list.htm>
- **主要 API / 能力**  
  - 淘宝客：商品、店铺、优惠券、转链、订单等。  
  - 千牛：卖家工作台、消息、订单等。  
  - 新零售、智慧门店、电商工具、无线应用等。
- **面向谁**  
  卖家、ISV、推广者、企业。  
- **与「控制用户手机淘宝」的关系**  
  不能控制用户手机上的淘宝 App；API 用于你的系统里查商品、做转链、管理卖家端等。

---

### 5. 京东 (JD)

- **开放平台 / 联盟**  
  - 京东联盟：<https://union.jd.com/>  
  - 京东秒送等：<https://opendj.jd.com/>
- **主要 API / 能力**  
  - 京东联盟：转链（商品/店铺/活动/优惠券）、商品搜索、分类、京粉精选、订单查询、推广位等。  
  - 其他开放能力：按业务线有不同 OpenAPI。
- **面向谁**  
  推广者、媒体、开发者、商家。  
- **与「控制用户手机京东」的关系**  
  不能控制用户手机上的京东 App；API 用于推广、转链、查订单等后台/推广侧能力。

---

### 6. 美团 (Meituan)

- **开放平台**  
  - 官网：<https://openapi.meituan.com/>  
  - 外卖开放：<https://open.waimai.meituan.com/>  
  - 企业用餐：<https://eps.meituan.com/open/>
- **主要 API / 能力**  
  - 外卖：订单查询、接单、订单管理、配送等。  
  - 企业用餐：部门/员工/订单管理等。  
  - 渠道接入、OpenH5 等。
- **面向谁**  
  商家、ISV、企业。  
- **与「控制用户手机美团」的关系**  
  不能控制用户手机上的美团 App；API 用于商家/企业侧系统对接。

---

### 7. 抖音 (Douyin) / 字节跳动

- **开放平台**  
  - 官网：<https://developer.open-douyin.com/>  
  - 服务商：<https://partner.open-douyin.com/>
- **主要 API / 能力**  
  - 用户：授权登录、用户信息、粉丝数据等。  
  - 内容：视频发布、视频数据、私信分享、朋友日常分享等。  
  - 载体：抖音小程序、移动应用、网站应用、直播小玩法、开放能力 SDK。
- **面向谁**  
  自营开发者、综合运营平台、代运营服务商。  
- **与「控制用户手机抖音」的关系**  
  不能控制用户手机上的抖音 App；API 用于「你的应用/小程序」里发视频、查数据、做登录等。

---

### 8. 小红书 (Xiaohongshu)

- **开放平台**  
  - 小程序开放：<https://miniapp.xiaohongshu.com/>  
  - 文档：<https://miniapp.xiaohongshu.com/doc/DC214380>
- **主要 API / 能力**  
  - 小程序：商品、订单、担保支付、客服、订阅消息等。  
  - 行业方案：酒旅、票务、小说等。  
  - 自研商家、服务商入驻与开发。
- **面向谁**  
  自研商家、服务商。  
- **与「控制用户手机小红书」的关系**  
  不能控制用户手机上的小红书 App；API 用于小红书小程序与商家/服务商系统对接。

---

### 9. 哔哩哔哩 (B站)

- **开放平台**  
  - 文档/接入：<https://bilibili.apifox.cn/> 等  
  - 社区整理的「野生 API」仅供学习，非官方控制接口。
- **主要 API / 能力**  
  - 账号授权、用户信息。  
  - 视频稿件：上传、编辑、删除、查询。  
  - 专栏、图片上传。  
  - 直播：开播、直播间消息（如 WebSocket）等。
- **面向谁**  
  创作者、应用开发者。  
- **与「控制用户手机 B站」的关系**  
  不能控制用户手机上的 B站 App；API 用于你的应用/工具里发视频、做直播相关能力。

---

### 10. 百度地图 (Baidu Map)

- **开放平台**  
  - 官网：<https://lbsyun.baidu.com/>  
  - 产品与文档：<https://lbs.baidu.com/products/direction> 等
- **主要 API / 能力**  
  - 地理编码 / 逆地理编码（HTTP，需 AK）。  
  - 路线规划：驾车、公交、步行、骑行、货车等。  
  - POI、定位、地图展示等（Web/移动 SDK）。
- **面向谁**  
  任何开发者（申请 AK 即可）。  
- **与「控制用户手机百度地图 App」的关系**  
  不能控制用户手机上的百度地图 App；但**可以用 HTTP API 在你自己的服务里做地理编码、路线规划**，再通过 Bridge 的「打开百度地图」深链接把结果在用户手机上用百度地图打开（若需要）。

---

### 11. 高德地图 (Amap)

- **开放平台**  
  - 官网：<https://lbs.amap.com/>  
  - Web 服务：<https://lbs.amap.com/api/webservice/guide/api/georegeo>、路径规划等
- **主要 API / 能力**  
  - 地理编码 / 逆地理编码（Web 服务 API，需 Key）。  
  - 路径规划：步行、公交、驾车等；高级路径、路径规划 2.0。  
  - POI、定位、地图展示等（Web/移动 SDK）。
- **面向谁**  
  任何开发者（申请 Key 即可）。  
- **与「控制用户手机高德地图 App」的关系**  
  同上：不能控制高德 App，但**可以用 HTTP API 做地理编码、路线规划**，再结合「打开高德地图」深链接在用户手机里展示。

## 三、在本项目中的可行用法

### 1. 社交 / 支付 / 电商 / 生活（微信、QQ、支付宝、淘宝、京东、美团、抖音、小红书、B站）

- **官方 API**  
  - 若你开发的是**自己的网站/App/后端**，需要「微信登录、支付宝支付、淘宝客转链、抖音发视频」等，用各平台开放平台即可。  
  - 这些 API **都不是**「在用户手机上控制对应 App」的接口。
- **Moltbot Bridge 侧**  
  - 已通过 **AppShortcutsApi** 支持**深链接打开**这些 App 或指定页面（如微信扫一扫、支付宝付款码、淘宝搜索等）。  
  - 无法通过官方 API 或 Bridge 读取用户在这些 App 内的聊天、订单、支付结果。

### 2. 地图（百度、高德）

- **官方 API**  
  - 百度：申请 AK，调用地理编码、逆地理、路线规划等 **HTTP API**。  
  - 高德：申请 Key，调用地理编码、逆地理、路径规划等 **Web 服务 API**。  
  - 可在**服务端或 Gateway 扩展**里请求这些 API，得到地址、路线、POI 等结果，再展示或下发给手机。
- **Moltbot Bridge 侧**  
  - 已有 **LocationApi** 使用系统能力做「设备当前/最后位置」与「地理编码/逆地理」（系统 Geocoder）。  
  - 若要做**路线规划、更丰富 POI、路况**，可**在 Gateway 扩展或单独服务中**接入百度/高德的 HTTP API（需用户或项目配置 AK/Key），与 Bridge 的「打开百度/高德地图」深链接配合使用。
